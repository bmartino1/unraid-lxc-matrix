==================================================
COMPONENT VERSIONS
==================================================

  Docs: https://element-hq.github.io/synapse/latest/

Element Web
  Source: GitHub releases (element-hq/element-web)
  Build script fetches latest release tarball

  EOL: Ongoing open-source

PostgreSQL
  Version: 16 (from postgresql.org APT repo)
  EOL: November 2028
  Notes: Must use C locale for Matrix Synapse

coturn
  Source: Debian bookworm packages (4.6.x)
  Docs: https://github.com/coturn/coturn

Nginx
  Source: Debian bookworm (1.22.x)
  Notes: Requires libnginx-mod-stream for SNI routing

Certbot
  Source: python3-certbot-nginx from Debian repos
  Auto-renewal: via cron every day at 3am

Debian OS
  Version: Debian 12 Bookworm
  EOL: June 30, 2028

==================================================
WHY DEBIAN 12 BOOKWORM (NOT BUSTER/BULLSEYE)
==================================================
- Buster (10): EOL August 2022 — DO NOT USE
- Bullseye (11): Only ships Java 11 — JVB2 works but Java 17 preferred
- Bookworm (12): Ships Java 17 natively, all repos support it, EOL 2028
- Trixie (13): ships with java 21 nativle Jitsu won't install

=> Bookworm is the correct choice


==================================================
TWO-PHASE SETUP MODEL
==================================================
Phase 1 (Build): createLXCarchive.sh runs on Unraid host.
  - Installs all packages into a temp Debian 12 LXC
  - No domain, no secrets, no config files written
  - Services disabled/stopped
  - setup.sh placed at /root/setup.sh inside the image
  - Packs container as matrix.tar.xz for GitHub release

Phase 2 (Setup): User runs ./setup.sh --domain chat.example.com
  - Runs inside the LXC after first boot
  - Writes all config files with real domain and generated secrets
  - Starts all services
  - Optionally gets Let's Encrypt certs

==================================================
PORTS USED
==================================================
External (forward to LXC):
  80/tcp   - HTTP (ACME challenge + redirect)
  443/tcp  - HTTPS (Nginx SNI stream router)

Internal (SNI Routing only):
  3478/udp - coturn TURN/STUN
  3478/tcp - coturn TURN/STUN (TCP)
  5349/tcp - coturn TURNS (TLS)
  10000/udp - Jitsi Video Bridge media

Internal (loopback only):
  8008     - Matrix Synapse HTTP
  8443     - Nginx internal HTTPS (SNI-terminated)
  9000     - Synapse metrics/health
  5280     - prosody BOSH + WebSocket
  5347     - prosody component port (jicofo/JVB)
  5222     - prosody c2s
  5269     - prosody s2s
  9090     - JVB Colibri HTTP
  5432     - PostgreSQL
  6379     - Valkey
  4443     - JVB TCP media relay

==================================================
JITSI WIDGET-ONLY SECURITY
==================================================
meet.<domain> is restricted by Nginx:
- Direct browser navigation -> HTTP 403
- Only accessible when Origin or Referer = Element domain
- Content-Security-Policy: frame-ancestors 'self' https://<element-domain>
- Element Web config points jitsi.preferred_domain at meet.<domain>

This means video calls work within Element but Jitsi cannot be used
as a standalone public meeting room (by design for private deployments).
Only when Patch scirpt is ran.
